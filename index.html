오토노미 스튜디오 (Autonomy Studio)

(주)퓨처드라이브 · 2023
TypeScript React Electron Recoil Webpack MUI Django Docker GitHub Actions
팀 구성 풀스택 개발자 3명, 자율주행 개발자 1명
담당 역할 프론트엔드 구조 개선, 시스템 성능 최적화, 빌드/배포 자동화 구축

개요

자율주행 차량 제어, 시뮬레이션 테스트, 디버깅 등 흩어져 있던 다양한 운영 도구를 단일 플랫폼으로 통합하여 제공하는 웹 기반 운영 서비스입니다.

주요 작업

  • Frontend UI 프레임워크 전환 및 컴포넌트 아키텍처 설계

    Django Template으로 작성된 코드가 복잡해지면서 화면을 수정하는 데 번거로움이 많았습니다. 이를 해결하기 위해 React로 만든 간단한 데모 화면을 직접 보여주며 마이그레이션 시의 장단점을 팀원들에게 설명하고 설득했습니다. 이후 UI 코드를 독립적인 React/TypeScript 환경으로 분리하여 백엔드와의 의존성을 낮췄습니다.

  • Frontend 초기 로딩 성능 최적화

    페이지 기능이 늘어나면서 첫 화면이 열리는 데 4초 이상 걸리는 문제가 발생했습니다. 이를 해결하기 위해 Webpack 코드 스플리팅을 적용하고, 정적 이미지와 폰트 파일을 압축한 뒤 WebP, WOFF2 확장자로 변환하여 번들 크기를 70% 줄였습니다. 결과적으로 첫 로딩 시간을 0.5초로 단축시켜 사용성을 개선했습니다.

  • Frontend 실시간 데이터 통신 구조 개편 (Polling → WebSocket)

    서로 다른 네트워크의 클라이언트와 브라우저 간 실시간 데이터 동기화 한계를 해결하고자 했습니다. 기존 Polling 방식의 조회를 Socket.io 기반의 양방향 통신 구조로 전면 재설계하고, 데이터 로딩 중 Skeleton UI를 적용하여 데이터 동기화 지연 시간을 2초에서 0.5초로 75% 단축했습니다.

  • ETC Git 브랜치 전략 설계 및 CI/CD 파이프라인 구축

    팀 내 브랜치 전략의 부재와 수동 배포 방식으로 인해 배포 리드타임이 15분 소요되고 월평균 5건의 장애가 발생했습니다. Git Flow 기반 브랜치 전략 설계와 온보딩 가이드를 작성해 팀에 전파하였고, GitHub Actions 기반 CI/CD 자동화로 수동 배포 과정을 전면 제거하여 배포 시간을 2분으로 단축하고 배포 장애를 월 5건에서 1건으로 감축했습니다.

성과

  • 초기 번들 사이즈 약 70% 감소 및 주요 페이지 로딩 속도 단축 (4초 → 0.5초)
  • 배포 리드 타임 단축 (15분 → 2분) 및 수동 배포 장애 감축 (월 5건 → 1건)
  • 실시간 데이터 동기화 지연 시간 75% 단축 (2초 → 0.5초) 및 불필요한 데이터 조회 제거

충전 트레일러 관제 서비스 (FMS)

(주)퓨처드라이브 · 2025
TypeScript React Recoil Socket.io Kakao Map API shadcn/ui Tailwind CSS v3 MSW
팀 구성 프론트엔드 개발자 1명, 백엔드 개발자 2명, 자율주행 개발자 4명, 오퍼레이터 2명
담당 역할 대시보드 설계 및 프론트엔드 개발

개요

자율주행 충전 트레일러의 효율적인 운영을 위해 실시간 이동 경로, 배터리 충전 현황, 예약 일정 등을 통합적으로 관리하는 대시보드 시스템입니다. 지도를 기반으로 여러 대의 트레일러 상태를 실시간 모니터링하고 원격으로 예약을 조율할 수 있습니다.

주요 작업

  • Frontend 상태 기반 UI 모델링 및 예약 프로세스 관리

    차량 모델에 따라 충전 단계와 조작 권한 제어가 상이하여 컴포넌트 내부에 조건부 분기 처리가 많아지는 문제가 있었습니다. 유한 상태 머신(FSM)을 도입해 예약 단계별(대기/이동/충전/완료) UI 상태를 데이터 객체로 구조화하여 대응했습니다. 컴포넌트 가독성을 대폭 향상시키고 신규 스펙 추가에 유연하게 대처할 수 있도록 설계했습니다.

  • Frontend 실시간 관제 데이터 시각화 및 최적화

    여러 대의 충전용 트레일러의 이동 경로 및 배터리 현황 데이터를 화면에 끊김 없이 실시간 모니터링해야 했습니다. Socket.io 기반 양방향 통신 구조를 설계하고 렌더링 최적화(Memoization)를 적용하였으며, Kakao Map API를 연동하여 차량 및 사용자의 실시간 위치를 부드럽게 시각화했습니다.

  • Frontend 독립적 개발 및 테스트를 위한 Mock API 구축

    물리 차량 통신 프로토콜 정의와 백엔드 API 개발 지연으로 프론트엔드 작업에 병목이 생겼습니다. 개발 초기 데이터 명세를 사전에 확정하고 MSW(Mock Service Worker)를 도입하여 모의 API 기반의 병렬 개발 환경을 구축함으로써 개발 프로세스 효율을 높였습니다.

성과

  • FSM 추상화 도입으로 복잡한 조건부 분기 코드 제거 및 컴포넌트 구조 단순화
  • API 의존성 제거를 통한 프론트엔드 병렬 개발로 프로젝트 전체 일정 2주 단축

기기 제어를 위한 실시간 원격 제어 환경 구축

담당 역할 원격 조작 인터페이스 및 비디오 스트리밍 웹 전반 설계/구현

개요

원격지에 있는 오퍼레이터가 웹 브라우저를 통해 자율주행 차량의 내부 화면을 실시간으로 확인하며, 키보드와 마우스로 차량의 소프트웨어 개발 환경을 직접 제어할 수 있도록 돕는 원격 솔루션입니다.

주요 작업

  • Frontend WebRTC 기반의 기기 원격 제어 기능 개발

    웹 브라우저와 단말기 간의 실시간 이벤트 동기화를 위해 WebRTC Data Channel을 사용해 키보드·마우스 이벤트를 송수신하는 인터페이스를 구현하고, 비디오 스트리밍 채널과 연동하는 React 컴포넌트를 설계했습니다.

  • Frontend 실시간 이벤트 전송 최적화 및 안정화

    원격 제어 조작 시 대량의 마우스 좌표 및 키 입력 이벤트가 동시 전송되면서 메인 스레드 연산 병목과 반응 속도 지연이 발생할 우려가 있었습니다. 입력 전송 루프에 Debounce 제어 로직을 구현해 입력 이벤트를 안정적인 주기로 전송하고 다운샘플링하여 조종 안정성을 높였습니다.

성과

  • WebRTC Data Channel 도입으로 원격 조종 반응 속도 및 화면 송출 지연 최소화
  • Debounce 필터 적용으로 프론트엔드 메인 스레드 병목 방지 및 제어 UX 안정성 확보

주요 작업

  • Frontend 한 손 조작이 가능한 모바일 예약 인터페이스 최적화

    기존에 사용하던 앱은 모바일 기기에서 예약을 진행할 때 여러 페이지를 이동해야 하는 번거로움이 있었습니다. 한 화면 내에서 예약 현황을 실시간으로 보며 한 손으로 범위를 드래그해 편리하게 예약할 수 있도록 모바일 예약 타임라인 및 바텀시트 UI를 설계했습니다.

    동시 예약 요청이 몰릴 때 동일한 시간대에 중복 예약이 발생하는 데이터 싱크 불일치 현상이 잦았습니다. Drizzle ORM 트랜잭션을 구현하여 데이터베이스 레벨에서 원자성을 검증해 중복 예약을 차단하고, 구글 캘린더 API를 연동하여 일정을 실시간 양방향으로 동기화했습니다.

성과

  • 기존 대비 예약 소요 시간 단축
